# 스마트 소비 분석기 (Smart Expense Tracker)

매달 반복되는 카드 이용 대금, 대출 상환 이자, 기타 가계 고정 지출 내역을 체계적으로 기록하고 관리하는 프리미엄 웹 애플리케이션입니다.

---

## 1. 프로젝트 목적
본 프로젝트는 개인의 매월 지출 세부 내역(신용카드, 대출금, 고정비 및 기타 비용)을 수동으로 일일이 계산하는 번거로움을 해결하기 위해 개발되었습니다. 직관적이고 반응이 빠른 구글시트 방식의 웹 인터페이스를 통해 지출을 입력하고, 입력 즉시 실시간 총합을 계산하며, 월별 소비 트렌드 변화를 한눈에 대시보드로 분석할 수 있도록 돕습니다.

## 2. 왜 이 프로젝트가 필요한가?
- **체계성 부재**: 매달 어디에 얼마를 쓰는지 정리가 안 되면 과소비로 이어지기 쉽습니다.
- **계산의 불편함**: 각 카드사 웹사이트나 뱅킹 앱을 여러 개 번역하며 금액을 수동 합산하는 것은 비효율적입니다.
- **데이터 흐름의 시각화**: 과거 지출 추이와 당월 카테고리별 비중을 직관적으로 시각화하여 불필요한 고정 지출을 줄일 수 있는 의사결정을 돕습니다.
- **안전한 데이터 보존**: 외부 클라우드나 DB에 개인 금융 정보가 전송되지 않고 브라우저 샌드박스 내부(`LocalStorage`)에 안전하게 저장됩니다.

## 3. 주요 기능
- **기본 및 사용자 정의 지출 항목 관리**: 현대카드, 롯데카드, 신한카드, 대출1(국민은행), 대출2, 기타 등의 기본 템플릿 항목을 제공하며, 사용자가 직접 추가/삭제할 수 있습니다.
- **실시간 통화 포맷팅 및 자동 합계**: 숫자를 입력하는 즉시 자동으로 세 자리마다 쉼표(`,`)가 표시되고, 하단 총 지출 금액 위젯이 실시간으로 동기화됩니다.
- **월별 저장 및 히스토리 관리**: 월별 입력 값을 독립적으로 저장하고 보존하며, 과거 기록을 클릭 한 번으로 간편하게 불러오거나 삭제할 수 있습니다.
- **데이터 시각화 대시보드 (Chart.js)**:
  - **월별 지출 트렌드 (Line Chart)**: 전체 누적된 월별 지출 합계를 꺾은선 그래프로 표시하여 소비 증감 추이를 확인합니다.
  - **지출 카테고리 비중 (Doughnut Chart)**: 당월 지출 금액을 가진 각 항목의 비율을 원형 그래프로 시각화해 지출이 편중된 곳을 쉽게 분석합니다.
- **프리미엄 글래스모피즘 UI & 듀얼 테마**: 모던한 다크모드와 라이트모드 전환 기능을 지원합니다.

## 4. 실행 방법 (단계별 상세 설명)
외부 서버나 복잡한 패키지 설치 없이 브라우저에서 바로 열어서 사용할 수 있는 정적 웹 앱(SPA)입니다.

1. 본 저장소의 코드를 내려받거나 폴더를 엽니다.
2. 폴더 내 `index.html` 파일을 더블 클릭하거나, 원하는 웹 브라우저(Chrome, Edge, Safari 등)로 엽니다.
3. 화면 상단에서 지출을 기록할 연도와 월(예: 2026년 06월)을 선택합니다.
4. 각 지출 항목에 해당하는 입력창에 이번 달 지출 금액을 숫자만 입력합니다. (숫자를 치면 자동으로 원화 3자리 콤마가 반영되며, 실시간으로 총액이 업데이트됩니다.)
5. 만약 기본 항목 외에 추가하고 싶은 항목(예: `통신요금`, `구독료`)이 있다면 상단의 **'항목 추가'** 입력란에 이름을 적고 추가 버튼을 누릅니다.
6. 입력이 완료되면 하단의 **'이번 달 지출 데이터 저장하기'** 버튼을 누릅니다. 저장과 동시에 과거 목록 및 그래프가 최신화됩니다.

## 5. 환경 설정 방법
- 이 프로젝트는 별도의 데이터베이스나 서버 세팅이 필요하지 않습니다.
- 최신 웹 브라우저(HTML5, CSS Grid, LocalStorage, ES6 Javascript 호환 브라우저)만 있으면 즉시 동작합니다.
- 외부 라이브러리(Chart.js 및 Lucide Icons)는 HTML 상에서 CDN 방식으로 자동 연동되므로 인터넷이 연결된 환경에서 구동하는 것을 권장합니다.

## 7. 보안 및 주의사항
- **개인정보 및 보안**: 본 앱은 사용자의 입력 데이터를 외부 서버나 API로 전송하지 않습니다. 모든 정보는 오직 현재 사용 중인 브라우저의 `LocalStorage` 영역에 로컬로 암호화되지 않은 형태로 저장됩니다.
- **데이터 유실**: 브라우저의 캐시(임시 파일)나 쿠키/사이트 데이터를 강제 초기화하면 로컬에 저장된 과거 지출 데이터가 영구적으로 삭제될 수 있으므로 주의하시기 바랍니다.

## 8. 테스트 방법 (임시 데이터 검증)
시스템이 완벽하게 동작하는지 검증하기 위한 임시 가짜 데이터를 활용한 테스트 시나리오입니다.

1. **테스트 데이터 주입**:
   - `index.html`을 연 다음, 월 단위를 `2026-05`로 변경하고 임의의 지출 금액(현대카드: 300,000 / 대출1: 500,000 / 기타: 100,000)을 입력하고 저장 버튼을 누릅니다.
   - 월 단위를 다시 `2026-06`으로 변경하고 현대카드에 `400,000원`을 입력하고 저장 버튼을 누릅니다.
2. **기능 검증**:
   - 우측 상단의 **'월별 지출 트렌드'** 차트에 5월 대비 6월의 소비가 증가하는 꺾은선이 올바르게 나타나는지 확인합니다.
   - **'지출 카테고리 비중'** 도넛 차트에 각 지출의 퍼센테이지가 마우스 호버 시 정상 표기되는지 검증합니다.
3. **테스트 데이터 초기화 (정리)**:
   - 검증이 끝난 후 우측 하단 **'과거 저장 목록'** 리스트에서 테스트로 생성했던 월(`2026-05`, `2026-06`) 항목의 우측 쓰레기통(삭제) 아이콘을 클릭하여 모든 테스트 데이터를 깨끗하게 지웁니다.
   - 이 과정을 통해 실운영 데이터에 영향을 주지 않고 안전하게 브라우저 저장소를 원래 상태로 보존할 수 있습니다.

## 9. 확장 가능성
- **클라우드 데이터 연동**: 향후 Firebase Auth 및 Firestore 데이터베이스 플러그인을 결합하여 로그인 기능을 추가하고 기기 간 데이터 실시간 동기화를 구현할 수 있습니다.
- **엑셀/CSV 내보내기**: 저장된 내역을 엑셀 파일(`.xlsx`)이나 쉼표 구분자 파일(`.csv`)로 다운로드할 수 있는 추출 기능을 추가해 별도의 회계 소프트웨어로 관리할 수 있습니다.
- **지출 예산 경보**: 월별 예산 한도를 설정하고, 실시간 입력 금액이 예산을 초과할 경우 경고 위젯 색상을 붉은색으로 동적 변화시키는 예산 알림 피드백을 추가할 수 있습니다.
