# 스마트 소비 분석기 (Smart Expense Tracker)

매달 반복되는 카드 이용 대금, 대출 상환 이자, 기타 가계 고정 지출 내역을 체계적으로 기록하고 관리하는 프리미엄 웹 애플리케이션입니다.

---

## 1. 프로젝트 목적
본 프로젝트는 개인의 매월 지출 세부 내역(신용카드, 대출금, 고정비 및 기타 비용)을 수동으로 일일이 계산하는 번거로움을 해결하기 위해 개발되었습니다. 직관적이고 반응이 빠른 구글시트 방식의 웹 인터페이스를 통해 지출을 입력하고, 입력 즉시 실시간 총합을 계산하며, 월별 소비 트렌드 변화를 한눈에 대시보드로 분석할 수 있도록 돕습니다.

## 2. 왜 이 프로젝트가 필요한가?
- **체계성 부재**: 매달 어디에 얼마를 쓰는지 정리가 안 되면 과소비로 이어지기 쉽습니다.
- **계산의 불편함**: 각 카드사 웹사이트나 뱅킹 앱을 여러 개 번역하며 금액을 수동 합산하는 것은 비효율적입니다.
- **데이터 흐름의 시각화**: 과거 지출 추이와 당월 카테고리별 비중을 직관적으로 시각화하여 불필요한 고정 지출을 줄일 수 있는 의사결정을 돕습니다.
- **안전한 데이터 보존**: 외부 클라우드나 DB에 개인 금융 정보가 전송되지 않고 브라우저 샌드박스 내부(`LocalStorage`)에 안전하게 저장되거나, 사용자가 지정한 본인의 개인 **구글 드라이브(Google Drive)**에만 백업됩니다.

## 3. 주요 기능
- **기본 및 사용자 정의 지출 항목 관리**: 현대카드, 롯데카드, 신한카드, 대출1(국민은행), 대출2, 기타 등의 기본 템플릿 항목을 제공하며, 사용자가 직접 추가/삭제할 수 있습니다.
- **실시간 통화 포맷팅 및 자동 합계**: 숫자를 입력하는 즉시 자동으로 세 자리마다 쉼표(`,`)가 표시되고, 하단 총 지출 금액 위젯이 실시간으로 동기화됩니다.
- **월별 저장 및 히스토리 관리**: 월별 입력 값을 독립적으로 저장하고 보존하며, 과거 기록을 클릭 한 번으로 간편하게 불러오거나 삭제할 수 있습니다.
- **구글 드라이브 클라우드 동기화**:
  - **드라이브 백업**: 로컬 브라우저에 임시 보관 중인 모든 지출 내역과 사용자 항목 스키마를 개인 구글 드라이브에 `smart_expense_data.json` 파일로 안전하게 백업합니다.
  - **드라이브 복원**: 기기를 변경하거나 브라우저 캐시가 비워진 상황에서도 구글 드라이브에 올려둔 백업 파일을 다시 가져와 즉시 데이터를 전체 복구합니다.
- **데이터 시각화 대시보드 (Chart.js)**:
  - **월별 지출 트렌드 (Line Chart)**: 전체 누적된 월별 지출 합계를 꺾은선 그래프로 표시하여 소비 증감 추이를 확인합니다.
  - **지출 카테고리 비중 (Doughnut Chart)**: 당월 지출 금액을 가진 각 항목의 비율을 원형 그래프로 시각화해 지출이 편중된 곳을 쉽게 분석합니다.
- **프리미엄 글래스모피즘 UI & 듀얼 테마**: 모던한 다크모드와 라이트모드 전환 기능을 지원합니다.

## 4. 실행 방법 (단계별 상세 설명)
외부 서버나 복잡한 패키지 설치 없이 브라우저에서 바로 열어서 사용할 수 있는 정적 웹 앱(SPA)입니다.

1. 본 저장소의 코드를 내려받거나 폴더를 엽니다.
2. **권장:** 폴더 안 `start.bat`을 더블클릭합니다. Node.js로 `http://localhost:5500` 서버가 뜨고 브라우저가 자동으로 열립니다. (구글 로그인·드라이브 백업을 쓰려면 반드시 이 방식. Node.js가 없으면 [nodejs.org](https://nodejs.org)에서 설치)
3. `localhost`와 `127.0.0.1`로 각각 저장했던 데이터는 서버의 `data/store.json`으로 자동 합쳐집니다. 예전 데이터가 안 보이면 `http://localhost:5500`을 한 번 연 뒤, 필요하면 `http://127.0.0.1:5500`도 새로고침하세요.
4. `index.html`을 직접 더블클릭해도 지출 입력/저장은 가능하지만, 구글 로그인은 동작하지 않습니다. 주소창이 `file://`로 시작하면 잘못된 실행입니다.
5. 화면 상단에서 지출을 기록할 연도와 월(예: 2026년 07월)을 선택합니다.
6. 각 지출 항목에 해당하는 입력창에 이번 달 지출 금액을 숫자만 입력합니다. (숫자를 치면 자동으로 원화 3자리 콤마가 반영되며, 실시간으로 총액이 업데이트되고 자동 저장됩니다.)
7. 만약 기본 항목 외에 추가하고 싶은 항목(예: `통신요금`, `구독료`)이 있다면 상단의 **'항목 추가'** 입력란에 이름을 적고 추가 버튼을 누릅니다.
8. 입력이 완료되면 하단의 **'이번 달 지출 데이터 저장하기'** 버튼을 누릅니다. 저장과 동시에 과거 목록 및 그래프가 최신화됩니다.

## 5. 구글 드라이브 연동 환경 설정 방법
안전한 개별 클라우드 동기화를 구현하기 위해, 사용자는 본인의 구글 클라우드 계정에서 API 키 정보를 받아 웹앱에 설정해야 합니다.

### [구글 클라우드 콘솔 설정 단계]
1. [구글 클라우드 콘솔](https://console.cloud.google.com/)에 접속하여 새 프로젝트를 생성합니다.
2. **API 및 서비스 > 라이브러리** 메뉴로 이동하여 **"Google Drive API"**를 검색한 뒤 **사용(Enable)** 버튼을 누릅니다.
3. **API 및 서비스 > OAuth 동의 화면**으로 이동합니다.
   - User Type을 **External(외부)**로 선택하고 앱 이름 및 지원 이메일을 입력합니다.
   - 범위(Scope) 추가 단계에서 `.../auth/drive.file` 스코프를 추가합니다. (이 스코프는 사용자가 허용한 전용 파일에만 앱이 접근할 수 있게 해주는 안전한 권한 수준입니다.)
   - 게시 상태를 **테스트** 단계로 유지한 상태에서 본인의 구글 이메일 주소를 **테스트 사용자**에 추가해야 테스트 로그인이 활성화됩니다.
4. **API 및 서비스 > 사용자 인증 정보** 메뉴로 이동합니다.
   - **사용자 인증 정보 만들기 > OAuth 클라이언트 ID**를 생성합니다.
     - 애플리케이션 유형을 **웹 애플리케이션**으로 선택합니다.
     - **승인된 자바스크립트 원본(Origin)**에 웹앱이 실행되는 주소를 입력합니다. (로컬 파일 주소 `file://` 대신 로컬 라이브 서버 주소 `http://localhost:5500` 또는 본인이 배포한 웹사이트의 도메인을 등록해 주어야 보안 정책에 걸리지 않고 작동합니다.)
     - 생성이 완료되면 생성된 **클라이언트 ID(Client ID)**를 복사합니다.
   - **사용자 인증 정보 만들기 > API 키**를 생성하고 복사합니다. (선택사항이나, 구글 드라이브 내부 검색 최적화를 위해 입력하는 것을 추천합니다.)
5. **웹앱 연동**:
   - 웹앱 하단의 **'API 연동 설정'** 링크를 클릭합니다.
   - 복사해 둔 **OAuth 2.0 클라이언트 ID**와 **API Key**를 입력한 후 저장 버튼을 누릅니다. (정보는 브라우저의 로컬 스토리지에 안전하게 기억되므로 매번 입력할 필요가 없습니다.)
   - 대시보드의 **'구글 로그인'** 버튼을 누르고 구글 계정을 선택하여 드라이브 접근 동의를 수행하면 동기화 기능이 활성화됩니다.

```text

 ├── server.js      # Node 로컬 정적 서버 + 공유 데이터 API
 ├── data/          # localhost·127.0.0.1 통합 저장소 (store.json)
 ├── index.html     # 애플리케이션 마크업 및 외부 라이브러리 연동
 ├── styles.css     # 프리미엄 다크/라이트 테마 정의 및 글래스모피즘 스타일링
 ├── app.js         # 데이터 저장, 계산, 구글 API 연동 및 Chart.js 렌더링 등의 제어 로직
 └── README.md      # 본 프로젝트 사용설명서 (한국어)
```

## 7. 보안 및 주의사항
- **개인정보 및 보안**: 본 앱은 사용자의 민감한 개인 금융 데이터를 개발자나 타사의 서버로 절대 송신하지 않습니다. 구글 드라이브 동기화 기능을 사용할 때도 오직 사용자가 지정한 구글 드라이브 루트 폴더에만 데이터를 직접 안전하게 송수신합니다.
- **데이터 백업 권장**: 로컬 스토리지 데이터는 브라우저 사이트 쿠키 및 캐시 삭제 시 소실될 위험이 크므로, 중요 데이터를 입력한 후에는 반드시 **'드라이브 백업'** 버튼을 눌러 클라우드에 백업을 보존하는 것을 권장합니다.

## 8. 테스트 방법 (임시 데이터 검증)
시스템이 완벽하게 동작하는지 검증하기 위한 임시 가짜 데이터를 활용한 테스트 시나리오입니다.

1. **테스트 데이터 주입**:
   - `index.html`을 연 다음, 월 단위를 `2026-05`로 변경하고 임의의 지출 금액(현대카드: 300,000 / 대출1: 500,000 / 기타: 100,000)을 입력하고 저장 버튼을 누릅니다.
   - 월 단위를 다시 `2026-06`으로 변경하고 현대카드에 `400,000원`을 입력하고 저장 버튼을 누릅니다.
2. **기능 검증**:
   - 우측 상단의 **'월별 지출 트렌드'** 차트에 5월 대비 6월의 소비가 증가하는 꺾은선이 올바르게 나타나는지 확인합니다.
   - **'지출 카테고리 비중'** 도넛 차트에 각 지출의 퍼센테이지가 마우스 호버 시 정상 표기되는지 검증합니다.
3. **구글 드라이브 동기화 검증**:
   - 구글 로그인을 마친 상태에서 **'드라이브 백업'** 버튼을 누릅니다.
   - 브라우저의 F12 개발자 도구의 Application 탭에서 LocalStorage를 강제로 비우고 새로고침하여 앱을 초기화합니다.
   - **'구글 로그인'**을 다시 진행한 뒤, **'드라이브 복원'** 버튼을 눌러 이전에 작성했던 5월과 6월의 지출 정보 및 차트가 완벽하게 복구되는지 검증합니다.
4. **테스트 데이터 초기화 (정리)**:
   - 검증이 끝난 후 우측 하단 **'과거 저장 목록'** 리스트에서 테스트로 생성했던 월(`2026-05`, `2026-06`) 항목의 우측 쓰레기통(삭제) 아이콘을 클릭하여 모든 테스트 데이터를 깨끗하게 지웁니다.
   - 이 과정을 통해 실운영 데이터에 영향을 주지 않고 안전하게 브라우저 저장소를 원래 상태로 보존할 수 있습니다.

## 9. 확장 가능성
- **자동 동기화**: 지출 데이터를 저장하거나 항목을 편집할 때 구글 드라이브에 실시간으로 자동 백업을 동기화하는 기능을 확장할 수 있습니다.
- **다중 계정 및 기기 지원**: 구글 드라이브의 백업 파일을 공유하거나 불러오는 과정을 고도화하여 부부나 연인 간에 지출 데이터를 공동 관리할 수 있는 가계부로 발전시킬 수 있습니다.
- **예산 대비 지출 경고**: 항목별로 월 목표 예산을 세우고 예산 달성률을 실시간으로 추적하여 초과 지출이 우려될 때 시각적인 경고 팝업을 띄우는 예산 제어 장치를 추가할 수 있습니다.
